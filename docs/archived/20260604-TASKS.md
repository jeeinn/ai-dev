## 关键原则

1. **架构简单化**：避免过度设计，switch 够用就不上接口。
2. **可配置化**：Prompt、模型、权限都通过配置管理。
3. **轻量安全可控**：不依赖 Docker，自实现软隔离。
4. **渐进式演进**：先只读型，再写入型，逐步增强。
5. **安全优先**：命令白名单、分支限制、PR 人工 review。
6. **工具化思维**：LLM 通过工具与代码库交互，而非一次性生成。
